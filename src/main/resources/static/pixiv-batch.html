批量导入单作品
导入格式：url | title 或 id | title
每行一条，例如：https://www.pixiv.net/artworks/12345678 | 示例标题
小说链接示例：https://www.pixiv.net/novel/show.php?id=12345678 | 示例标题
仅 ID 示例：12345678 | 示例标题
仅写数字 ID 时默认按插画解析（等同于 https://www.pixiv.net/artworks/{id}）；若需要按小说解析，请在该行之前加一行 novel: 作为区段头。
区段头 artwork: / novel: 单独成行（大小写不敏感，全/半角冒号均可）；其下方的所有「仅 ID / id | title」按该类型解析，直到遇到下一个区段头或文本结束。明确的链接始终按链接自身类型解析，与所在区段无关。
标题可留空；下载前会自动获取真实标题。兼容 One-Tab，N-Tab 等标签页管理插件导出格式。
也兼容下方“导出全部”“导出未下载”按钮生成的作品列表，可直接重新导入。

⏰ 存为计划任务
按当前模式来源与上方全部下载 / 筛选设置创建一个后台自动下载的计划任务（仅管理员）。在 User / Search / 系列模式下配置好后保存即可。
从上次运行算起，每隔这么多分钟自动运行一次（例如 1440 = 每天一次，60 = 每小时一次）。
使用 Spring 的 6 位格式「秒 分 时 日 月 周」（含秒，比常见的 5 位 Unix Cron 多出最前面的秒位）。示例：0 0 3 * * * = 每天 03:00:00；0 0 */6 * * * = 每 6 小时；0 30 8 * * 1 = 每周一 08:30:00。
