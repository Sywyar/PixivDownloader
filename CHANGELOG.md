## [Unreleased]

### Features
- 下载插画作品或小说时，会额外在作品目录保存一个 `{作品ID}.meta.json` 元数据文件，记录从 Pixiv 抓取到的原始结构性信息（逐页尺寸与原图地址、真实发布时间、是否原创、官方标题 / 简介翻译、约稿信息等；不含收藏 / 浏览等计数与登录账号相关字段，小说还会去掉已另存的正文与内嵌图），供后续功能免重新联网即可读取；同时把「发布时间」与「原创标记」一并落库为可查询字段。这些信息全部取自下载时本就已抓到的数据，不会为此对 Pixiv 发起任何额外请求。该元数据文件被视为作品自身的一部分：随作品移动 / 删除而同步迁移 / 清除，并不计入配额打包与小说导出。此为后台基础能力，暂无界面直接展示。
- `config.yaml` 新增「内置功能模块开关」配置 `plugins.<插件 id>.enabled`（`plugins.gallery.enabled`、`plugins.novel.enabled`、`plugins.duplicate.enabled`，默认全部启用），可单独关闭对应内置功能模块。「下载工作台」是必选插件（下载页、下载执行与计划任务调度属核心使用路径），无法关闭、不提供该开关（即便手工把其开关写成 `false` 也会被忽略）。关闭某模块后该模块的页面、导航入口、API 与其维护任务都不再注册（访问其页面 / API 会返回 404），其在后台运行的服务也一并停用；但核心数据照常写入——下载历史、作品元数据、标签 / 作者 / 系列关系、小说正文、图片 Hash 等均不受任何插件开关影响，也不会删除数据库或文件，重新启用即可基于既有数据恢复。这些开关也可在 GUI 配置页新增的「插件」标签页用复选框编辑，每项展示对应插件注册时声明的名称与简介。修改后需重启服务生效。（统计「stats」已改为外置插件、不在内置功能模块开关之列：需作为插件包放入工作目录 `plugins/` 后由程序加载，安装后在下述「插件管理」页查看与热管理，未安装时其统计页面、API、静态资源、i18n 与导航都不注册。）
- 新增（仅管理员）插件管理后端 API：可经 HTTP 查询全部插件（内置 / 外置 / 必选但未安装）的状态、来源、运行期阶段、是否受管 / 必选、当前可用动词，以及对核心 API 的版本要求与对其它插件的依赖声明（`GET /api/plugins/status`）；并对外置插件施加运行期生命周期动词 `load / start / quiesce / stop / unload / reload`（`POST /api/plugins/{插件 id}/{动词}`）。内置插件不支持运行期热启停、未激活或未知插件、必选插件被停用等都会被守卫明确拒绝，失败响应同时带稳定机器码（与界面语言无关、便于程序判别）与本地化说明。该接口仅管理员可用，并在恢复模式下同样可用、便于在必选插件缺失时查询状态并驱动修复；它只做运行期管理，不涉及插件的安装 / 升级 / 删除。
- 新增（仅管理员）「插件管理」页面（顶部导航栏新增「插件管理」入口）：以卡片列表展示全部插件（内置 / 外置 / 必选但未安装）的名称、状态、来源、版本、运行期阶段、对核心 API 的版本要求、插件间依赖与诊断说明；对受管的外置插件可直接按其当前可用动词执行运行期生命周期操作（加载 / 启动 / 静默 / 停止 / 卸下 / 重载），内置插件随主程序编译、只读展示不可热管理，操作成功后即时刷新状态与导航入口。页面消费上述插件管理 API，支持中英文切换与深色模式。

### Bug Fixes
- 修复在下载页清除队列后，各模式预览列表（快捷获取 / User / Search / 系列）中作品卡片的「✓ 已在队列」标记不会随之消失、残留过期状态的问题：此前「清除队列」只刷新 Search 预览、移除单个队列项也漏刷新「快捷获取」预览，现在队列的新增 / 移除 / 清除都会统一刷新全部四种模式预览的在队列标记。
- 修复删除作品 / 小说时，若部分文件因被占用或权限不足删除失败，会留下「一部分文件已删、数据库记录仍在」的残缺状态（作品在画廊里裂图，只能再点一次删除补完）的问题：现在文件删除改为原子操作——先把待删文件暂存到 `data/delete-staging/` 并记录一份恢复清单，再删除原文件，任一文件删除失败即回滚到删除前状态（作品文件原样复原、数据库不变），删除要么全部完成、要么完全不变，不会再让作品半损坏。若进程在删除中途意外退出，下次启动会据恢复清单把已删文件从暂存复原（无法据清单恢复的暂存目录则保留并记录日志，供人工恢复），不会再误删唯一的备份而把半删除状态永久化。

## [v1.13.0] - 2026.6.11

### Features
- 计划任务支持按任务指定单独的代理与 Cookie：「存为计划任务」卡片新增「设置单独的代理」「设置单独的cookie」复选框（cookie 输入框带「使用当前保存的cookie」快捷填入），计划任务管理页原「授权 Cookie」按钮改为「指定单独的 代理/cookie」弹窗，可随时为任务绑定 / 替换 / 清除独立的 HTTP 代理（host:port）与 Cookie。设置后该任务每轮运行中对 Pixiv 的全部访问（发现、元数据、图片 / 小说下载、站内信过度访问检测）都改走指定代理与 Cookie；未设置的项沿用默认行为（全局代理设置 / solo 模式自动绑定的 Cookie），取消勾选并保存会在确认后果后清除对应设置。
- 画廊删除作品 / 小说改为保留「删除标记」：通过画廊（单个、批量、导出后删除）删除时仍会删除磁盘文件与派生数据，但历史记录保留删除标记，之后批量下载 / 油猴脚本 / 计划任务不会再把这些作品当作未下载而重新下载，跳过原因显示为「已经下载过，但被删除」。批量下载页与各油猴脚本的下载设置新增「允许已删除的作品被重新下载」复选框（默认不勾选），勾选后这些作品视为未下载、重新下载成功后删除标记自动清除；该设置同样会随「存为计划任务」快照保存，旧版本创建的计划任务在启动时自动补齐该设置（默认不勾选）。被标记删除的作品对画廊、筛选、统计等所有历史读取入口不可见。
- 添加 Moonlight 主题支持
- 插画画廊和小说画廊批量管理增强：支持按当前筛选结果全选，批量删除、加入收藏夹或导出；导出弹窗可选择打包方式（按作者 / 作品分类或以作品 ID 为文件夹名）、打包格式与是否导出后删除源文件，压缩包内含记录作品信息与原始路径的 manifest。
- 画廊、小说画廊、系列、统计页侧边栏收起按钮改为外凸的「导航」按钮，并在其下新增「任务列表」按钮（仅管理员）：点击后侧边栏内容切换为任务队列，展示后台压缩任务的作品级打包进度（已处理 / 总作品数进度条）与剩余有效期，完成后可在列表中手动下载，点击导出后自动打开；再次点击收起为窄条，仅显示各任务的简要状态与进度，悬浮其上可查看详细信息。
- 新增「约稿作品」下载支持：批量下载页 User 模式作品类型新增「约稿」一项，输入画师 ID 或粘贴约稿页链接（`https://www.pixiv.net/users/{id}/request/artworks`，会自动切到该项）即可预览并下载该画师已完成公开的约稿成品；「快捷获取」新增「我的约稿作品」一键拉取本账号的约稿成品；管理员还可把上述两种来源「存为计划任务」，按周期自动发现并下载新约稿（约稿成品即普通插画，沿用既有下载、判重与文件命名）。

### Changed
- 下载根目录为相对路径时，数据库中的目录记录改为跟随软件目录解析：整个软件文件夹被复制 / 搬迁到新位置后，画廊与下载历史可直接定位文件，不再出现「文件夹搬走了但数据库找不到文件」；旧记录在启动时自动迁移为新格式。GUI「迁移下载目录」对当前下载根目录的改写相应增强：新目录仍在软件目录内且选择同步配置时，仅把 `config.yaml` 写为相对路径、记录继续跟随软件目录；迁出软件目录或不同步配置时按原有方式固定为绝对路径记录。
- 多人模式 never-delete / timed-delete 的「已下载直接返回成功」判定不再命中被画廊删除的作品：这些作品文件已不在磁盘，现在视为未下载放行重新下载。
- GUI 管家以 jar 方式（`java -jar`）启动时，更新横幅的「立即更新」不再下载并启动 exe 安装包（该流程仅适用于安装版 / 便携版 exe），改为提示并在确认后打开对应版本的 GitHub 发布页，引导手动下载；以 exe 启动时行为不变。

### Bug Fixes
- 修复「我的收藏」「珍藏集」计划任务设了「首次抓取上限」后只下载最新一批、更早的积压永远不再补下的问题：此前已下载的作品也会占用每轮额度，稳定排序下表头那批已下载作品每轮都把额度吃满、窗口停在原地；现在已下载的作品免费跳过、不占额度，每轮真正下满该来源中尚未下载的 N 个作品、分多轮逐步把积压抽干（被筛选条件跳过的作品仍占额度，以限住每轮对 Pixiv 的元数据请求量）。
- 修复计划任务绑定的 Cookie 因失效被自动降级为受限（匿名）模式后，仍保留原 Pixiv 账号标识的问题：此前该任务虽已转为无 Cookie 的受限模式、不再使用该账号凭证，却仍会被同账号其它任务触发的过度访问暂停一并冻结，并在账号级恢复与相关通知中按原账号归类；现在自动降级（以及手动解除单独 Cookie）时会一并清除该任务的账号绑定与该账号的「无视风险」放行记录。
- 修复多页作品部分页面下载失败（如网络中断重试耗尽）时仍被写入下载历史的问题：此前队列虽提示失败，但数据库已记录该作品，再次下载会被「跳过已下载」判重挡住，缺失的页面无法补齐；现在任一页面（含动图）下载失败都不写入下载历史与统计，队列明确提示「部分图片下载失败」，直接重新下载即可补齐全部页面。同样地，批量下载开启「实际目录检测」时，从磁盘反向恢复记录不再接受页号不连续的文件集合（如缺第 0 页的部分失败残留），这类作品视为未下载、重新下载即可补齐。
- 修复打包 / 导出压缩包在构建中途失败时残留半成品 zip 的问题：此前打包失败仅把任务标记为出错，已部分写入的 zip 既不会被运行期清理也不会被过期清理删除（只能等下次启动时统一清理），长时间运行的实例会逐渐堆积无用文件；现在构建失败会立即删除该残留文件。

## [v1.12.0] - 2026.6.6

### Features
- 批量下载与计划任务的「小说设置」新增「新下载小说自动翻译」：开启后，新下载完成的小说会自动用 AI 翻译为指定目标语言（可设分段字数），系列还会自动生成该语言的译文合订本；翻译固定使用该小说 / 系列的默认名词映射表。翻译在后台进行、不阻塞下载，下载队列与计划任务队列会显示「AI 翻译中 / 等待前系列章节翻译完成」等状态；下载本身成功即视为成功，翻译失败只在该项标注、不影响已下载文件。仅管理员可用，且需先配置 AI；翻译结果可在小说详情页切换内容语言查看。
- 小说详情页「听书」新增「富感情朗读（多角色）」引擎（beta，可用但尚不稳定）：由大语言模型把整章逐句归属到说话人，每个角色用固定音色逐句合成、连续播放并高亮当前段落与字幕。首次播放前可选择分段粒度、角色音色「花名册」与旁白基调，支持为角色生成或上传参考音做音色克隆，并可查看 / 编辑各角色音色、处理音色冲突。语音合成可选 VoxCPM（自建）及小米 MiMo、CosyVoice、Fish Audio、MiniMax、ElevenLabs、阿里 Qwen3-TTS、字节豆包 / Seed-TTS 等多家云端 / 自建 TTS 引擎，无需自备 GPU 也可使用云端引擎。仅管理员可用，需在后端同时配置文本模型与朗读引擎（任一未配置则不显示该引擎选项）。
- 作品详情页新增收藏按钮，可将作品加入 / 移出本地收藏夹（与小说详情页一致），支持勾选已有收藏夹或快速新建收藏夹。
- 计划任务的自动挂起通知现在除邮件外，还会并行推送到已配置的推送通道。
- 计划任务新增三类运行结果通知（邮件与推送并行）：绑定的 Cookie 失效但该任务无需 Cookie 时，本轮会自动降级为受限模式继续运行、成功后通知一次；整轮运行失败时通知（仅状态由非失败转入失败时发送一次，连续失败不重复）；运行成功且本轮有新下载时发送一条结果摘要。
- 计划任务自动挂起 / 过度访问暂停的通知（邮件与推送）携带更完整的信息：过度访问暂停会逐条列出被挂起的计划任务名称与 ID；鉴权失效 / 熔断的通知会附带任务类型、触发方式与恢复提示（这类任务已挂起、需人工重新授权后才会继续，故不展示下次运行时间）；作品自动重试达上限的通知会附带任务类型、触发方式、下次预定运行时间与该作品的 Pixiv 直链（该作品所属任务并未挂起，仍按计划继续运行）。
- GUI 通知配置页新增「需要通知的类型」开关：可按通知类型（过度访问暂停、登录失效挂起、失败熔断、重试耗尽、降级匿名续跑、运行失败、运行摘要）分别开关，取消勾选的类型不再通过任何介质（邮件与推送）发送，默认全部启用。
- 新增代理自动配置（PAC）端点 `/proxy.pac`（仅本机可访问）：在系统 / 浏览器的「自动代理配置脚本 URL」处填入该地址，即可让网页版 Pixiv 仅对 Pixiv 相关域名走后端配置的同一个代理、其余流量直连，无需再开启系统级代理；代理配置变更会自动反映到 PAC 内容。
- GUI 管家新增主题选择（跟随系统 / 浅色 / 深色），即时生效并写入 `config.yaml` 的 `app.theme`，下次启动保留；选择「跟随系统」时会监听操作系统的浅 / 深色变化并重涂界面。
- GUI 管家新增「迁移下载目录」：可把数据库中记录的下载 / 分类目录路径逐个改写为新路径，用于已手动把文件搬到新位置后修正记录（仅修改数据库记录，不会移动磁盘上的文件，留空则保持原路径，新目录须为已存在的文件夹）；若改写的是当前下载根目录，会提示是否一并同步 `config.yaml` 的下载根目录，避免新作品仍下载到旧目录。

### Changed
- 批量下载设置中的「收藏到」（收藏夹选择）不再被记忆：每次打开 / 刷新页面都默认为「不加入收藏夹」，需在本次会话内重新选择；以此创建「存为计划任务」时仍会按当时所选的收藏夹快照保存。
- 后端未配置 AI 文本模型时，小说 / 系列详情页不再显示「AI 翻译」入口（此前即便未配置也会显示，点击后才报错）。
- GUI 管家的 AI 配置整合为单个「AI 模型」页，用顶部「模态」下拉在「文本模型」与「TTS 模型（beta）」间切换，两者同时保存并生效。
- 计划任务绑定的 Cookie 被检测失效、但该任务的下载设置 / 范围无需 Cookie 时，现在会自动清除该失效 Cookie 快照并将任务转为受限模式（无 Cookie）继续运行，避免每轮再用失效 Cookie 反复探测、重复通知；之后如需抓取 R-18 / 受限内容需重新授权 Cookie。
- 计划任务「首次抓取上限」改为按进入本轮队列的作品数计算，已下载跳过、筛选跳过或单作品失败的队列项也会占用额度。

### Bug Fixes
- 修复计划任务管理卡片对已挂起的任务（Cookie 失效 / 过度访问暂停 / 手动暂停）仍显示一个并不会自动到来的「下次运行时间」的问题：这类任务需人工重新授权或恢复后才会继续，卡片不再展示具体的下次运行时间，而是提示需人工恢复。
- 修复创建 / 编辑计划任务失败时只笼统提示「保存失败」、看不出具体原因的问题：现在会显示后端返回的实际原因（如「Cron 表达式无效」），可对照表单里的 Cron 格式说明更正；恢复账号、授权 Cookie 失败时同样显示具体原因。

### Changed
- 小说详情页改为完全基于本地数据展示，不再访问 Pixiv：正文取自本地保存的原始内容，封面与内嵌插图只用本地已下载的文件，缺失的资源静默跳过不显示，因此断网或无法访问 Pixiv 时也能正常阅读已下载的小说。打开一本本地未下载的小说时，与图片详情页一致，仅显示「该小说尚未下载到本地」提示，不会自动联网拉取。
