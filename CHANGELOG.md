## [Unreleased]

### Features
- GUI 新增「首页」单步引导向导：「服务已就绪 → 配置 → 开始下载 → 浏览画廊 → 完成」每一步是独立页面，点击「下一步 / 完成」或后端信号自动推进时以滑动动画切换（服务状态带实时呼吸点）。第二步可直接在 GUI 内完成首次配置（设置管理员账号并选择 solo/multi 模式），无需再打开网页 setup；引导会随用户在浏览器打开下载页、完成画廊操作指引而自动推进，画廊指引完成后 GUI 窗口自动回到前台继续后续步骤。整套引导未全部完成前每次启动都停留在首页，完成后默认回到「状态」页；完成页附油猴脚本与动图 ffmpeg 等可选进阶说明。
- 下载页与画廊页新增操作指引遮罩：首次访问自动以高亮聚光逐步介绍关键操作（填 Cookie、选择下载方式、开始下载、查看进度、浏览画廊等），画廊指引最后会提示回到程序窗口继续引导；左下角常驻「操作指引」按钮可随时重新查看，支持中英文与深色模式。
- 漫画画廊与小说画廊在没有任何已下载作品时，主列表会显示「没有下载的作品，快去下载页下载吧！」提示并附带一个跳转到下载页的按钮（仅在无搜索/筛选条件时显示；有筛选但无结果仍显示原有的「没有匹配的作品」）。
- 漫画画廊与小说画廊的顶部搜索框新增搜索范围选择：可在综合（标题或作者名）、标题、作者名、作品/小说 ID、作者 ID、简介、标签（模糊）、标签（精确）之间切换，ID 与标签精确按精确匹配，选择会被记忆并在两类画廊间同步；搜索框提示文字会随所选范围变化，搜索无结果时搜索框与筛选按钮会变红提示。
- 新增「Pixiv 体验增强工具箱」油猴脚本，并纳入 All-in-One 与网页端脚本安装列表；当前提供两项功能：
  - 「已下载作品加边框」：对服务器已下载过的作品与小说在缩略图上加边框，可按需开关并配置边框样式（需本机 localhost、solo 模式且已登录）。
  - 「一键获取 Cookie」：一键读取当前 Pixiv 网站 Cookie 并保存到服务器，与批量下载页的「Cookie」设置共用同一存储（需 solo 模式且已登录，服务器地址不限）。
- 「页面批量下载器」的队列标记（缩略图复选框，点击加入/移出队列）现在同时覆盖插画与小说卡片；已加入队列的项还可在缩略图上叠加一圈边框，便于一眼分辨哪些已被抓取，边框可在设置中开关（默认开启），并可像体验增强工具箱一样自定义宽度、颜色与线型（实线/虚线/双线）。
- 油猴脚本全面支持小说下载：
  - 「URL 批量导入单作品下载器」可识别小说单作品链接并作为队列项下载，下载进度/状态与网页端 `pixiv-batch` 对齐，并新增小说格式（TXT/HTML/EPUB）设置
  - 「单作品图片下载器（Java后端版）」在小说页面可直接通过后端下载当前小说，支持格式选择
  - 「User 批量下载器」「页面批量下载器」新增「插画+漫画+动图 / 小说」切换，可批量下载画师全部小说或当前页面小说，并提供小说格式设置；新增「系列生成合订本」开关与独立的合订本格式选择（默认并推荐 EPUB），批量结束后会按所属系列各生成一份合订本
  - 「单作品图片下载器（Local Download）」新增纯本地小说下载，直接在浏览器内生成包含内嵌图片与封面的 EPUB 电子书
- 「单作品图片下载器（Java后端版）」与「单作品图片下载器（Local Download）」面板新增收起按钮与 Mini 悬浮球（堆叠在体验增强工具箱悬浮球下方），收起/展开状态会在页面刷新和站内导航后记忆，并与其它脚本面板共享互斥，可通过油猴菜单重新打开面板
- 批量下载页（solo 模式）Cookie 栏新增「一键导入 Cookie」按钮：点击后自动打开 Pixiv 页面，由「体验增强工具箱」读取当前登录 Cookie 回传服务器并自动关闭，几秒内完成、无需手动复制粘贴；若尚未安装工具箱，会弹出聚光式引导，引导期间页面其它内容不可点击，必须先点击高亮的「体验增强工具箱」安装按钮才能继续。引导的安装步骤额外提供「我已安装 All-in-One」按钮，点击后直接进入获取流程。读取登录态需要脚本管理器能返回 HttpOnly 会话 Cookie（PHPSESSID）；少数浏览器/管理器环境无法暴露该 Cookie，此时会明确提示并取消保存（不会覆盖已有 Cookie）；Cookie 区域也常驻一条指引链接，导入无效时可按《获取 Cookie 指南》改用 Cookie-Editor 手动粘贴。
- 批量下载页新增 Cookie 清除入口；solo 模式退出登录时会同步清除服务器端保存的 Cookie。
- Windows GUI 在线更新下载安装包时新增实时进度条，下载期间会暂时阻止界面语言切换以避免打断更新流程。
- 批量下载页 Search 模式的「附加筛选」新增「标签(精确匹配)」「标签(模糊匹配)」两个条件：逗号分隔可填多个标签，需全部命中（精确=与作品标签完全相等，模糊=被作品标签包含），两框同时填写则两者都需满足，对插画与小说均生效。
- 批量下载页 Search 区在「插画+漫画+动图 / 小说」下方新增「搜索模式 / 作品批量获取模式」切换。作品批量获取模式复用关键词、内容、搜索方式、排序与全部附加筛选，可指定起始页与结束页批量抓取并跨页去重，抓取结果在本地按 60/页 分页浏览，并提供「获取」「将此页内容加入队列」「将全部结果加入队列」三个操作（multi 模式下单次抓取页数仍受配额上限约束，solo 与管理员不限）。

### Changed
- 批量下载页 Cookie 相关提示（保存/清除/一键导入等结果）改为显示在 Cookie 区域，不再出现在底部下载队列状态栏，避免与下载进度混在一起。
- GUI 模式下首次启动不再自动弹出浏览器打开网页 setup 页面，首次配置改为在 GUI「首页」引导内完成；nogui / headless（含 `--no-gui`）部署仍会像以前一样自动打开 `/setup.html`，该页也保留为后备入口。
- 所有油猴脚本面板（单作品 Java 后端版 / 单作品 Local / URL 批量导入 / User 批量 / 页面批量 / 体验增强工具箱）展开时与各自 Mini 悬浮球保持同一垂直高度，看起来从悬浮球处展开。
- 根目录与网页端脚本列表中的独立油猴脚本统一采用功能后缀文件名（`User Batch`、`Page Scrape`、`URL Batch`、`Local Download`、`Toolbox`），All-in-One 也同步包含体验增强工具箱。
- 根目录发布的油猴脚本新增 `@updateURL` / `@downloadURL` 元数据，指向 GitHub raw 文件，脚本管理器可直接检查并下载更新。
- 通过批量下载页安装油猴脚本时，`@updateURL` / `@downloadURL` 会改为当前后端的 `/api/scripts/{id}.user.js` 安装地址，避免安装后仍从 GitHub raw 源检查或下载。
- 「URL 批量导入单作品下载器」聚焦单作品：不再解析或下载小说系列链接，仅处理插画/漫画/动图/小说单作品
- 小说下载队列状态细化为正文、内嵌图片、封面、保存、收藏、归档等阶段，并在批量下载页展示正文/封面字节进度和内嵌图片数量进度。
- 通过后端下载小说为 EPUB（含系列合订 EPUB）时，正文中的上传图片现在会一并打包进 EPUB 文件并在阅读器中直接显示，不再只显示 `[uploadedimage:id]` 占位文字；封面也会内嵌为 EPUB 封面页（阅读器书架可显示封面缩略图）。
- 小说 EPUB 改为符合 EPUB3 规范的结构：新增标准导航目录文档，正文按章节（`[chapter:]`）拆分为可跳转的目录项（系列合订本为「小说 → 章节」两级目录）；同一小说/系列重新下载视为同一本书（阅读进度/书架不再重复）；并补充简介、上传日期、标签、Pixiv 来源链接与系列归组等电子书元数据。
- 小说系列合订本格式改为独立可选项（不再跟随单章下载格式）：批量下载页与「User 批量下载器」「页面批量下载器」均新增专门的「合订本格式」下拉，默认并推荐 EPUB（界面标注推荐并说明可内嵌封面/插图、生成多级目录、带书籍信息可在阅读器书架显示），仍可选 TXT/HTML；单章文件的下载格式（TXT/HTML/EPUB）与合订本格式互不影响。
- GUI 错误弹窗改为统一提示，并提供打开 HTML/TXT 日志按钮；详细错误进入日志，避免长错误或外部命令输出乱码挤进弹窗。
- GUI 状态页启动超过 10 秒时会显示已等待秒数，并提示该阶段可能正在更新数据库。
- Windows 安装包在安装前会检测 `PixivDownload.exe` 是否仍在运行，并提示关闭后重试。

### Security
- GUI 本地管理接口（`/api/gui/**`）新增一次性本地令牌校验；Swing GUI 会自动携带令牌，外部页面不能只凭 localhost 访问这些接口。

### Bug Fixes
- 修复批量下载页「清除队列」只清理前端队列、不终止后端图片/动图/小说下载任务的问题；多人模式访客只会清理自己的任务。
- 修复小说下载遇到 Pixiv Cookie 失效或 Pixiv 上游错误时只返回笼统失败的问题，现在会显示更明确的登录 Cookie 或上游 HTTP 错误提示。
- 修复登录页暗色模式显示不完整，以及小说阅读页切换语言后动态元数据、标签、系列导航和收藏状态不刷新的问题。
- 修复 GitHub Pages 静态预览站点无法切换语言的问题。
- 修复 `--intro` 启动时可能跳转到非标准介绍页的问题。
- 修复 Windows 安装完成后以管理员上下文启动应用的问题，安装后会按当前用户启动。
- 修复 GUI 开机自启动在安装路径或 Startup 路径包含空格时创建快捷方式失败的问题，并修复 PowerShell 错误输出在中文系统中乱码的问题。
- 修复打包后的预发布版本号丢失 `-rc` 等后缀的问题。
- 修复 Windows 安装包升级安装后，安装目录的 `app` 文件夹下会残留旧版本 JAR 文件、长期占用磁盘空间的问题；现在升级时会先清理旧的程序文件再写入新版本（用户数据与配置不受影响）。
- 修复在线更新检查忽略预发布后缀的问题：此前 `1.0.0-rc.1` 与 `1.0.0` 被视为同一版本，运行预发布版本的用户既不会被提示升级到对应正式版，也不会被提示同阶段的更新版本。现在按 `n.n.n-后缀.n` 比较，后缀优先级由低到高为 未知后缀 < snapshot/nightly < dev/alpha < beta/m/preview < rc < 无后缀正式版，同阶段再按后缀数字比较。

### Security
- 收紧 solo 模式下未登录可访问的静态资源范围，仅保留入口页、登录/引导页及其必要依赖；访客邀请页面仍可加载画廊和小说页所需共享资源

### Bug Fixes
- 修复 GitHub Pages 预览站点的引导页资源路径问题，避免样式或脚本在 Pages 环境中加载失败
- 修复 Windows 安装包卸载后残留注册表信息导致再次安装被误判为已安装、升级时进入维护页，以及升级/修复流程重复触发 FFmpeg 下载的问题

- 修复登录页和引导页在权限收紧后部分资源访问、重定向保留和访客邀请页面资源加载异常的问题

### Build
- 发布流程支持手动创建指定标签的草稿 Release；正式 JAR、Windows 安装包和正式 Release 仍只在版本标签推送时构建发布
- Windows 运行时镜像补充字符集与本地化模块，并为打包程序显式使用 UTF-8 标准输出和错误输出

## [v1.8.1] - 2026-05-15

### Features
- Windows GUI 新增在线更新能力：启动后可静默检查新版本，状态页可手动检查；发现新版时显示更新横幅，并可下载、校验安装包后启动安装程序
- GUI 配置页新增“在线更新”配置组，支持配置更新清单 URL、启用/关闭更新检查以及启动后自动检查
- GUI 配置页支持部分配置保存后热重载，代理、多用户配额与限流、下载扁平目录、登录限流、维护开关、HTTPS 域名和在线更新配置无需重启即可应用；界面会标注“热重载”或“需重启”
- 监控页接入浅色/暗色主题切换，统计图会随主题刷新；主题切换会同步到其他已打开页面

### Changed
- GUI 状态页默认窗口尺寸增大，状态页支持滚动，并将页面跳转与功能操作分组展示，减少 FFmpeg 状态、更新横幅和操作按钮在小窗口中被截断的问题
- 批量下载页的油猴脚本安装卡片适配浅色/暗色主题，安装与源码按钮显示更一致

### Security
- setup 初始化、GUI 状态/重启/配置热重载/更新、迁移和维护等本地管理端点新增 Origin 校验，防止外部网页借浏览器访问 `localhost` 发起副作用请求

### Build
- 发布产物新增在线更新清单，供 GUI 在线更新功能识别最新版本和安装包校验信息

### Removed
- Release 不再发布 Windows 便携 zip 包；Windows 用户请使用安装包，仍可按 README 开发指南在本地构建便携版

### Documentation
- README.md 与 README_en.md 重整项目简介、安装说明和截图入口，并拆分浅色/暗色模式截图文档

## [v1.8.0] - 2026-05-14

### Features
- 新增 Pixiv 小说下载能力，支持通过小说 ID、小说详情页 URL 和小说系列 URL 获取元数据并下载为 TXT、HTML 或 EPUB，下载记录会持久化小说标题、作者、标签、分级、系列顺序、字数、页数、封面和原始 Pixiv 标记内容
- 新增本地小说画廊 `pixiv-novel-gallery.html` 与小说阅读页 `pixiv-novel.html`，支持小说列表、搜索筛选、分页、封面展示、作者入口、系列导航、上一章/下一章跳转和本地内容阅读
- 新增小说系列合订功能，可将已下载的同系列章节按顺序合并导出为 TXT、HTML 或 EPUB，单章文件仍会保留
- 批量下载页支持按 Pixiv 用户、搜索结果以及漫画/小说系列 URL 加入下载队列；系列模式支持分页预览、当前页入队和全部分页补齐入队
- 收藏夹支持同时包含作品和小说，小说下载完成后可按选择的收藏夹自动归档，并在作者视图中统计和展示小说相关内容
- 小说内容渲染支持 Pixiv 常见标记，包括分页、章节、注音、跳转链接、Pixiv 图片引用和上传图片引用；小说内嵌上传图片会随内容下载并在本地页面中展示
- URL 批量导入 userscript 支持识别 Pixiv 小说和小说系列链接，并提交到后端小说下载流程
- 本地系列目录页 `pixiv-series.html` 支持漫画/小说双模式，展示系列封面、简介、标签、章节顺序、阅读时长和当前章节定位，并可从详情页、画廊或系列列表互相跳转
- 图片画廊与小说画廊新增漫画/小说顶部切换、全部视图入口、侧栏状态记忆、筛选与分页状态记忆，以及分页省略处输入页码跳转
- 邀请码可为小说单独配置可见标签和作者范围，并支持批量删除过期邀请码
- 多标签页语言切换会自动同步到其他已打开页面

### Changed
- 批量导入相关命名统一为“批量导入单作品”，独立 userscript 文件名同步更新；旧 localStorage 与队列来源会自动兼容迁移

### Security
- 收紧本地管理接口访问判定，setup、GUI、迁移、维护等本地接口不再仅凭远端地址判断本地访问；首次初始化必须来自本地请求
- 多人模式用户只能查看和控制自己的下载状态、取消操作与 SSE 进度流；管理员与 solo 模式仍保留全局视图
- 下载子目录与落盘路径会拒绝不安全路径片段，降低目录穿越风险
- Pixiv 简介 HTML 会清理不安全链接和标签，访客访问下的收藏夹、历史、统计、小说作者/系列/标签等数据也会按邀请可见范围过滤

### Performance
- 优化访客模式下历史、统计和小说画廊的可见性过滤，减少大列表场景的额外处理开销
