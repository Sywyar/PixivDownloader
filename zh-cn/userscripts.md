# 油猴脚本

油猴脚本让你在 Pixiv 网页上直接操作——浏览时看到想下载的作品，点一下就加入队列，无需切换到 PixivDownloader 页面。

?> 油猴脚本是可选的。不安装脚本也可以通过 `pixiv-batch.html` 完成绝大多数下载操作。

---

## 前提条件

1. 浏览器安装了 [Tampermonkey](https://www.tampermonkey.net/) 扩展
2. PixivDownloader 后端正在运行
3. Solo 模式下需已登录

---

## 脚本列表

| 脚本名称 | 用途 |
|---------|------|
| **All-in-One 整合包** | 包含下面所有脚本（推荐安装这一个） |
| 页面批量下载器 (Page Scrape) | 在搜索页、排行榜等页面批量勾选下载 |
| User 批量下载器 (User Batch) | 在画师主页一键批量下载全部作品 |
| URL 批量导入 (URL Batch) | 在任意页面打开面板，粘贴链接批量下载 |
| 单作品下载器 (Java 后端版) | 在单作品页面一键下载（经后端） |
| 单作品下载器 (Local Download) | 在单作品页面一键下载（浏览器本地，**无需 Java 后端**） |
| 体验增强工具箱 (Toolbox) | 已下载作品加边框标记、一键导入 Cookie 等 |

---

## 安装方式

### 方式一：从 Web 管理页安装（推荐）

1. 启动 PixivDownloader，打开 `http://localhost:6999/pixiv-batch.html`
2. 找到顶部「**🧩 油猴脚本**」卡片，点击展开
3. 点对应脚本的「**⬇ 安装**」按钮
4. Tampermonkey 会弹出安装确认页，点「安装」

?> 通过这种方式安装时，脚本会自动将 `@connect` 配置为当前后端地址。

### 方式二：从 Release 下载

1. 前往 [Releases](https://github.com/Sywyar/PixivDownload/releases) 下载脚本文件（`.user.js`）
2. 将文件拖入 Tampermonkey 管理面板，点「安装」

## 各脚本使用说明

### All-in-One 整合包（推荐）

安装此包即可获得所有功能。如果你已单独安装了部分脚本，安装 All-in-One 后请禁用原来的独立版，**避免双面板冲突**。

### 页面批量下载器 (Page Scrape)

在 Pixiv **搜索页**、**关注动态页**、**排行榜页**等列表页面生效：

- 每个作品缩略图角落出现**复选框**
- 勾选想要的作品
- 点浮动面板上的「开始下载」

已加入下载队列的作品会在缩略图上显示边框标记。

### User 批量下载器 (User Batch)

在 Pixiv **画师主页**（`/users/<id>` 及其子页面）显示浮动面板：

- 一键将该画师的**全部作品**加入下载队列
- 支持插画和小说分开

### URL 批量导入 (URL Batch)

在**任意 Pixiv 页面**显示浮动面板，功能等同于 `pixiv-batch.html` 的「批量导入单作品」Tab：
- 粘贴 URL 列表批量下载

### 单作品下载器 (Java 后端版)

在**单个作品页面**（`/artworks/<id>`）显示下载按钮：
- 一键通过后端下载，包含元数据、自动收藏等功能

### 单作品下载器 (Local Download)

在**单个作品页面**显示下载按钮：
- 直接通过浏览器下载，**不需要 Java 后端**
- 没有元数据记录，不入画廊数据库

### 体验增强工具箱 (Toolbox)

在 Pixiv **任意页面**的右侧显示工具栏（各功能可单独开关）：

| 功能 | 说明 |
|------|------|
| **已下载作品加边框** | 服务器数据库里有的作品，缩略图上会加彩色边框，方便区分哪些已下载 |
| **一键导入 Cookie** | 一键读取 Pixiv Cookie 并保存到服务器（需开 Tampermonkey 特殊权限） |

## 部署在非 localhost 时的额外配置

脚本默认连接 `http://localhost:6999`。如果你的后端部署在其他机器或其他端口：

1. Tampermonkey 管理面板 → 找到对应脚本 → 点「编辑」
2. 修改脚本顶部的 `@connect` 行，将 `localhost` 替换为实际的服务器地址

通过 Web 管理页安装的脚本已经自动配置好了当前后端地址。

## 常见问题

### 安装了脚本但 Pixiv 页面上看不到任何变化

1. 确认 Tampermonkey 已启用（浏览器扩展图标显示脚本数量）
2. Tampermonkey 管理面板里确认脚本状态为「已启用」
3. 刷新 Pixiv 页面（脚本在页面加载时注入）
4. 确认当前页面与脚本的 `@match` 规则匹配（例如 User Batch 只在 `/users/` 页面显示）

### All-in-One 和独立脚本同时启用，出现两个面板

禁用独立版本，只保留 All-in-One，或反之。两个版本的存储数据是互相独立的，建议只用一个。

### 脚本提示「后端未运行」或「连接失败」

确认 PixivDownloader 后端服务正在运行，且脚本里配置的地址和端口正确。
