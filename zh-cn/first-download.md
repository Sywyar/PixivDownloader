# 第一次下载

本教程以最常见的场景为例，带你完成第一次下载的全过程。

## 前提

- PixivDownloader 已安装并启动
- 已完成[首次配置](/zh-cn/first-setup)

---

## 新手引导（首次自动）

首次以管理员身份打开下载页（Solo 模式 / 多人模式已登录管理员）时，会自动开始一个**跨页面的新手引导**，一步步带你走完从下载到浏览的完整流程：

1. 先问你想用的**称呼**，并检测后端到 Pixiv 的**网络连通性**（显示往返延迟，不通会提示检查代理 / 网络）
2. 逐区域认识下载页的 **Cookie 区 / 油猴脚本区 / 下载方式（Tab）**，并以「批量导入单作品」为例
3. 带你**粘贴示例作品链接 → 加入队列 → 了解附加筛选与下载设置 → 开始下载**，下载过程中会高亮状态栏与下载队列、实时显示进度
4. 下载完成后自动跳转**画廊**逐区域讲解（浏览视图 / 搜索 / 筛选 / 作品网格），再进入**作品详情页**讲解作品图 / 功能区 / 简介 / 作者 / 系列与相关

引导可随时点「跳过」，也能通过下载页右下角的「**操作指引**」按钮重新查看（已设置过称呼时会跳过称呼这一步、直接从连通性检测开始）。多人模式访客不会触发该引导。

?> 你在引导里填写的**称呼**会保存下来，用于画廊 / 小说画廊侧边栏底部的用户卡片，以及计划任务通知、邮件测试等系统邮件的问候语；留空时统一以「管理员」称呼。

下面的步骤是同一流程的手动版——无论跟着引导走还是自己操作，都能完成第一次下载。

---

## 第一步：打开下载页面

在浏览器中访问：

```
http://localhost:6999/pixiv-batch.html
```

**Solo 模式**下会弹出登录框，输入你在首次配置时设置的管理员账号和密码。

?> 如果不记得地址，打开 GUI → 状态页 → 点「批量下载」快捷按钮。

---

## 第二步：获取 Pixiv Cookie（推荐，非必须）

Cookie 是你的 Pixiv 登录凭证，配置后可以：

- 下载 R-18 / R-18G 内容（未配置时这类作品会跳过）
- 使用「搜索下载」和「快捷获取」功能
- 下载需要登录才能看到的内容

**不配置 Cookie 也可以下载全年龄公开作品。** 如果你只想先试试，可以跳过这一步，之后再补配置。

### 获取步骤

下面任选一种方式获取 Cookie，得到的内容都能用，按自己熟悉程度选一个即可。无论哪种方式，都请先在浏览器中登录 [Pixiv](https://www.pixiv.net/)。

?> 后端真正需要的只有 `PHPSESSID` 这一项（你正常登录后 Pixiv 下发的 `PHPSESSID` 形如 `12345678_xxxxxxxx`）。带上完整 Cookie 也可以，但只填 `PHPSESSID` 同样能用。

#### 方式一：Cookie-Editor 扩展（最简单）

1. 安装 [Cookie-Editor](https://chromewebstore.google.com/detail/cookie-editor/hlkenndednhfkekhgcdicdfddnkalmdm) 浏览器扩展（Chrome/Edge 均可）
2. 点击 Cookie-Editor 扩展图标
3. 右下角点 **Export** → 选择 **Netscape** 格式
4. 点「Copy Export」复制全部内容
5. 填入时格式选 **Netscape**

#### 方式二：开发者工具 → Application → Cookies

1. 在 Pixiv 页面按 **F12** 打开开发者工具
2. 切到 **Application（应用）** 面板 → 左侧 Storage → **Cookies** → 选中 `https://www.pixiv.net`
3. 在右侧列表里找到 **Name** 为 `PHPSESSID` 的那一行，复制它的 **Value**
4. 拼成 `PHPSESSID=刚复制的值` 一行（例如 `PHPSESSID=12345678_xxxxxxxx`）
5. 填入时格式选 **Header String**

?> 想一次带上更多 Cookie 时，可把列表里多行拼成 `名字=值; 名字2=值2` 的形式，同样用 **Header String** 格式。`PHPSESSID` 是 HttpOnly Cookie，但 Application 面板能正常显示它的值，所以这种方式无需调整任何油猴权限。

#### 方式三：开发者工具 → Network 抓请求头

1. 在 Pixiv 页面按 **F12** 打开开发者工具 → 切到 **Network（网络）** 面板
2. 刷新一下页面，在请求列表里点任意一条发往 `www.pixiv.net` 的请求（如顶部的 document 文档请求或某个 ajax 请求）
3. 在 **Headers（标头）** 里找到 **Request Headers（请求标头）→ `Cookie`** 这一项，复制它的完整值
4. 填入时格式选 **Header String**

?> 请求头里的 `Cookie` 已经是 `名字=值; 名字2=值2` 的格式，里面就包含 `PHPSESSID`，直接整段粘贴即可。

#### 方式四：油猴「一键导入 Cookie」（无需手动复制）

需先安装「**体验增强工具箱 (Toolbox)**」油猴脚本：

1. 在 `pixiv-batch.html` 的 Cookie 卡片里点「**一键导入 Cookie**」
2. 脚本会自动打开 Pixiv 页面读取 Cookie，并直接回传给后端，无需手动复制粘贴、也无需选格式

!> **此方式需要油猴授予 HttpOnly Cookie 读取权限。** Pixiv 的登录凭证 `PHPSESSID` 是 HttpOnly Cookie，油猴脚本默认读不到，因此一键导入会提示「未检测到 PHPSESSID」而失败。需到 **Tampermonkey（油猴）→ 设置 → 把配置模式设为 `高级` → 安全 → 将「允许脚本访问 Cookie」设为 `全部 / All`** 后再使用。

!> **不建议长期保持该设置为 `全部`**：一旦开启，你安装的**其它任何油猴脚本也能读取到你的登录凭证**，存在账号被盗风险。除非你信任已安装的全部脚本，否则建议**仅在使用一键导入时临时设为 `全部`，成功获取后立即改回 `除 HttpOnly 之外的全部 / All except HttpOnly`**。若不想动这个权限，优先用上面的方式二、方式三（开发者工具）——它们同样能拿到 HttpOnly 的 `PHPSESSID`，且无需改任何油猴权限。

### 填入 Cookie

> 以下手动粘贴步骤适用于方式一 / 二 / 三；方式四（一键导入）会自动回传，无需手动填入。

1. 在 `pixiv-batch.html` 页面找到顶部的「**Cookie**」卡片（灰色折叠卡片），点击展开
2. 把格式切换为与获取方式对应的项：方式一（Cookie-Editor 导出）选 **Netscape**；方式二、方式三（开发者工具）选 **Header String**
3. 把复制的内容粘贴进去
4. 点「**保存 Cookie**」

?> Cookie 有效期较长，通常几个月才需要重新获取一次。Pixiv 重新登录或修改密码后 Cookie 会立即失效。

## 第三步：下载一张作品

### 在 Pixiv 找到想下载的作品

在 Pixiv 上找到你想下载的作品，复制浏览器地址栏里的 URL，例如：

```
https://www.pixiv.net/artworks/12345678
```

或者只复制数字 ID 部分（`12345678`）也可以。

### 粘贴并下载

1. 在 `pixiv-batch.html` 顶部切换到「**🎨 批量导入单作品**」Tab
2. 在文本框里粘贴 URL（每行一条，可以一次粘多个）
3. 点「**获取信息**」— 等待几秒，会显示作品预览
4. 确认信息无误后，点「**开始下载**」

---

## 第四步：查看下载进度

下载开始后，页面下方「**下载队列**」区域会出现任务条目：

| 状态 | 含义 |
|------|------|
| ⏳ 等待中 | 排队等待开始 |
| 🔄 下载中 | 正在下载文件 |
| ✅ 已完成 | 下载成功 |
| ⏩ 已存在，跳过 | 该作品之前已经下载过，无需重复 |
| ❌ 失败 | 下载出错，点条目可查看原因 |

---

## 第五步：找到下载的文件

下载完成后，文件保存在**程序工作目录**下的 `pixiv-download/` 文件夹：

```
(程序目录)/
└── pixiv-download/
    └── 123456/          ← 作品 ID 命名的子目录
        ├── 123456_p0.jpg    ← 第 1 张图
        ├── 123456_p1.jpg    ← 第 2 张图（多图作品）
        └── ...
```

**程序目录在哪里？**

- **Windows 安装包**：安装时选的目录，默认 `C:\Users\用户名\PixivDownload\`
- **JAR 包**：你运行 `java -jar` 命令时所在的那个文件夹
- **GUI 方式**：在 GUI → 状态页 → 「下载目录」栏里有显示

?> 可以在 `config.yaml` 里修改 `download.root-folder` 改变保存位置，支持绝对路径。

---

## 第六步：在画廊里浏览

打开画廊页面查看所有已下载的作品（支持搜索、筛选、收藏等）：

```
http://localhost:6999/pixiv-gallery.html
```

详细用法见[作品画廊](/zh-cn/gallery)。

## 常见问题

### 点「获取信息」后报错

- **「需要 Cookie」**：该作品需要登录才能查看，请先配置 Cookie（见第二步）
- **「代理连接失败」**：检查代理是否正常运行，以及 `config.yaml` 里代理地址和端口是否正确
- **「作品不存在」**：作品可能已被作者删除或设为私有

### 下载完成但找不到文件

查看 `config.yaml` 里的 `download.root-folder` 配置项，它决定文件保存位置。如果配的是相对路径，实际位置是「程序工作目录 + 该路径」。

### 想批量下载更多内容

- 一次粘贴多个 URL → 见[URL 批量下载](/zh-cn/batch-download)
- 下载某位画师的全部作品 → 见[画师批量下载](/zh-cn/user-download)
- 按关键词搜索下载 → 见[搜索下载](/zh-cn/search)
