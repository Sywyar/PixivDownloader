# 计划任务

计划任务让后端按设定的时间间隔**自动发现并下载**新作品，无需每次手动操作。

**典型应用场景：**
- 关注了一位画师，想自动下载他今后发布的所有新作品
- 定期抓取某个搜索关键词的最新结果
- 自动跟踪一个漫画/小说系列的更新
- 定期同步你的收藏 / 珍藏集 / 已关注用户的新作（从「[快捷获取](/zh-cn/quick-access)」创建）

?> 计划任务仅管理员可使用。如果你是访客，请让管理员操作。

---

## 前提条件

1. **已登录管理员账号**
2. **已配置含 `PHPSESSID` 的 Cookie**（部分内容需要登录态）

!> 没有绑定 Cookie 的任务以「匿名模式」运行，只能下载全年龄公开内容，且无法访问你的私有关注/收藏。如需下载 R-18 内容或完整关注列表，必须先配置好 Cookie。

---

## 创建第一个计划任务

计划任务**不是单独填表创建**的，而是把你当前的下载配置「快照」下来作为任务。具体步骤如下：

### 以「关注画师」为例

**第 1 步：进入画师下载模式**

在 `pixiv-batch.html` 切换到「**👤 User 模式**」Tab。

**第 2 步：填入画师来源**

在输入框粘贴画师主页链接，例如：
```
https://www.pixiv.net/users/123456
```
也可以只填数字 ID（`123456`）。

**第 3 步：配置下载设置（可选）**

展开「下载设置」卡片，根据需要调整：
- **文件名模板**：默认 `{artwork_id}_p{page}`
- **收藏夹**：下载到哪个收藏夹
- **最大并发数**：一次同时下载几个作品（默认跟随全局设置）
- **作品间隔**：每个作品之间等多少毫秒（建议 500–2000，避免频繁请求 Pixiv）

**第 4 步：配置附加筛选（可选）**

展开「**附加筛选**」卡片，按需设置：
- 内容分级：只下载全年龄，还是包含 R-18？
- 排除 AI 生成作品？
- 限定特定标签？

**第 5 步：存为计划任务**

向下滚动到页面底部，找到「**⏰ 存为计划任务**」卡片：

1. 填写**任务名称**（方便区分，例如「画师 Sywyar 每日更新」）
2. 选择**触发方式**：
   - **固定周期**：例如 `60` 分钟执行一次
   - **Cron 表达式**：精确指定时间，例如每天凌晨 3 点 → `0 0 3 * * *`
3. （可选）设置**首次抓取上限**：限制首次运行抓取的作品数量，避免一次性下完整个历史（详见下文「[首次抓取上限](#首次抓取上限)」）；默认 `0` = 全量
4. 点「**创建任务**」

?> 创建成功后页面会弹出提示，切换到「⏰ 计划任务」Tab 可以看到刚创建的任务。

## Cron 表达式格式

PixivDownloader 使用 Spring 的 **6 位 Cron 格式**（比常见的 5 位多了最前面的「秒」位）：

```
秒  分  时  日  月  周
0   0   3   *   *   *   → 每天 03:00:00
0   30  8   *   *   1   → 每周一 08:30:00
0   0   */6 *   *   *   → 每 6 小时
0   0   9   1   *   *   → 每月 1 日 09:00
```

| 字段 | 取值范围 | 特殊值 |
|------|---------|-------|
| 秒   | 0–59 | `*`=每秒, `/n`=每n秒 |
| 分   | 0–59 | `*`=每分, `/n`=每n分 |
| 时   | 0–23 | `*`=每小时, `/n`=每n小时 |
| 日   | 1–31 | `*`=每天 |
| 月   | 1–12 | `*`=每月 |
| 周   | 0–7 | 0/7=周日, `*`=每天 |

## 来源类型的工作方式

> 除 User / Search / 系列外，「收藏 / 我的作品 / 关注新作 / 珍藏集」这几类直接在「[快捷获取](/zh-cn/quick-access)」Tab 展开对应来源后，由页面底部的「⏰ 存为计划任务」卡片创建。

### 画师新作（User 模式）

- **首轮**：下载该画师所有符合筛选条件的未下载作品（可能较多）
- **后续**：使用「水位线增量」——每轮只扫描到上次已处理的最新作品为止，只补新发布的，不重扫全部历史
- 效果：首轮可能下载几百张，之后每轮只下载新发的几张

### 搜索关键词（Search 模式）

- 按配置的页数抓取最新搜索结果
- 按「最新」排序且页数设为 `-1` 时，同样使用水位线增量，每轮只补最新的
- 效果：定期拉取该关键词下的新作品

### 漫画/小说系列（系列模式）

- 每轮列出系列全部章节，跳过已下载的，补充新章节
- 适合跟踪连载中的漫画或小说系列

### 我的收藏 / 我的作品 / 关注新作 / 珍藏集（从「快捷获取」创建）

在「⚡ 快捷获取」Tab 展开对应来源后（收藏 / 我的作品 / 关注新作，或点进某位关注画师 / 某个珍藏集），页面底部的「⏰ 存为计划任务」卡片即可创建：

- **我的收藏**：定期发现收藏夹（公开 / 不公开、插画 / 小说）里尚未下载的作品并补下
- **我的作品**：等同于按你自己账号的「画师新作」
- **已关注的用户的新作**：定期下载关注画师的最新插画/漫画/动图
- **珍藏集**：列出珍藏集内的插画与小说成员，跳过已下载的补下（**插画与小说都会下载**）

!> 这几类来源属于**账号私有**，对应任务必须已授权含 `PHPSESSID` 的 Cookie，否则会标记为「登录态失效」并暂停（不会以匿名模式空跑）。它们没有独立的模式标签页，**编辑时来源为只读**——要更换来源请删除后重新创建。

## 首次抓取上限

对于可能一次性下完大量历史的来源，可在「⏰ 存为计划任务」卡片填写**首次抓取上限**，限制抓取数量、降低触发 Pixiv「过度访问」警告的风险。`0` 表示全量抓取（不限）。

行为按来源是否有可靠的「最新」次序分两类（页面提示会自动只显示与当前来源匹配的那一条说明）：

- **首轮封顶**（画师新作 / 已关注用户的新作 / 「最新排序 + 结束页 -1」的搜索）：**仅首次运行**最多抓取最新的 N 个作品，水位线随即推进到最新；之后每轮只增量追新，更早的历史不再补下。适合「只想要最近 N 个、之后持续跟新」。
- **每轮上限**（我的收藏 / 珍藏集 / 非最新排序的「结束页 -1」搜索）：这些来源没有可靠的「最新」次序，因此按**每轮上限**执行——每次运行最多抓取 N 个新作品，分多轮逐步把积压抽干。**若希望最终完整下载全部作品，把上限设为 `0` 即可**（每轮不限）。

系列下载与固定页数搜索本身范围有界，不显示该选项。

!> 把上限设为 `0`（全量）时，首次运行可能一次性下完该来源的全部历史；作品很多时仍有触发过度访问警告甚至封号的风险，保存时页面会弹出确认。

## 管理已有任务

切换到「**⏰ 计划任务**」Tab，可以看到所有任务。

每个任务卡片右上角有**状态灯**：

| 颜色 | 状态 | 含义 |
|------|------|------|
| 🔘 灰 | 等待 / 已停用 / 已暂停 | 任务正常待机，或被手动暂停 |
| 🟢 绿 | 运行中 / 上次成功 | 任务正常 |
| 🟡 黄 | 排队中 | 多个任务同时到期，正在排队等待依次执行 |
| 🔴 红 | 失败 / Cookie 失效 / 被中断 / 过度访问暂停 | 任务需要关注，点卡片查看详情 |

### 常用操作

| 操作 | 说明 |
|------|------|
| **立即运行** | 不等定时，马上执行一次 |
| **授权 Cookie** | 绑定当前 Cookie 给该任务使用 |
| **启用 / 停用** | 暂时停止某任务的定时触发 |
| **手动暂停 / 恢复** | 与「停用」类似，但会在状态灯上区分显示 |
| **查看快照** | 查看任务创建时保存的来源和筛选配置 |
| **编辑** | 修改任务配置（重新走创建流程） |
| **待重试** | 查看本任务中失败等待重试的作品 |
| **删除** | 彻底删除任务（已下载的文件不受影响） |

## Cookie 授权

?> 如果你在 Solo 模式下**创建任务时**已有可用 Cookie，任务会自动绑定，无需手动授权。

如果任务显示「受限模式」，说明还没绑定 Cookie：

1. 确保 `pixiv-batch.html` 顶部的 Cookie 卡片里已保存含 `PHPSESSID` 的 Cookie
2. 在任务卡片上点「**授权 Cookie**」
3. 授权成功后任务切换为正常模式

Cookie 失效（Pixiv 重新登录或改密码后）时：
- 任务会自动停止并标记「登录态失效」（红灯）
- 同时发送邮件通知（需先配置 SMTP）
- 重新获取 Cookie 并保存后，再次点「授权 Cookie」即可恢复

## 过度访问暂停

当 Pixiv 检测到你的账号请求过于频繁时，会发送「过度访问」站内信警告。PixivDownloader 会自动检测并暂停**同账号的所有计划任务**，避免账号被封禁。

此时任务管理 Tab 顶部会出现横幅，提供两个恢复选项：

1. **「无视风险，立即恢复」** — 把本次警告标记为已放行，马上继续（仍有风险）
2. **「N 分钟后继续」** — 冷却一段时间再恢复（最少 60 分钟，默认 60 分钟）

建议选第 2 项，等待一段时间再继续。

---

## 邮件通知

计划任务在以下情况会发送邮件通知：
- Cookie 失效（登录态失效）
- 检测到过度访问
- 熔断挂起（连续失败次数达到阈值）
- 失败作品重试达到上限

!> 邮件通知需要先在 GUI「配置 → 邮件 / SMTP」配置 SMTP 账号，否则通知会静默失败。

## 相关配置

计划任务的相关配置在 GUI「配置 → 计划任务」或 `config.yaml` 的 `schedule.*` 下：

| 配置项 | 说明 | 默认值 |
|--------|------|-------|
| `schedule.enabled` | 总开关 | `true` |
| `schedule.tick-interval-ms` | 检查到期任务的间隔（毫秒），**修改需重启** | `60000` |
| `schedule.max-tasks` | 最多可创建的任务数 | `100` |
| `schedule.inbox-check-every` | 每下载多少个作品检测一次过度访问站内信 | `500` |
| `schedule.auth-failure-circuit-breaker` | 连续失败多少次触发熔断挂起 | `5` |
| `schedule.pending-max-attempts` | 失败作品最多自动重试几次 | `5` |
| `schedule.overuse-defer-default-minutes` | 过度访问暂停后「稍后继续」的默认分钟数（最少 60） | `60` |

详细配置说明见[配置参考](/zh-cn/configuration)。
